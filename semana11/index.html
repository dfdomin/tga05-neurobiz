📋 Memorando NeuroBiz

Rol inmersivo: Analista de Datos Estratégico. Tu misión consiste en traducir la estructura de datos de esta semana en una decisión de negocio medible para NeuroBiz S.A.S., empresa que coordina terapias para población neurodivergente con foco en TDAH, TEA y ACCC.

El equipo clínico informó que comparar el progreso de un paciente entre sesiones separadas exige avanzar y retroceder con rapidez. Cuando el historial solo se mira en una dirección, la revisión longitudinal consume tiempo y se pierde contexto terapéutico. Se solicita construir una lista doblemente enlazada para navegar el historial hacia adelante y hacia atrás con trazabilidad.

En cada evidencia usa, al menos, un KPI del caso: sesiones_asistidas, sesiones_fallidas, tasa_retencion, indice_progreso, nivel_hiperfoco y reconocimiento_patrones. La estructura no se estudia de forma aislada: se justifica por su impacto en la operación.

📚 Teoría aplicada con fuentes reales

Fuentes base para esta semana: Cairó & Guardati (2006)Joyanes Aguilar (2014)Weiss (2014)

Nodo con dos vínculos

Según Cairó Battistutti y Guardati Bueno (2006), una lista doblemente enlazada incorpora un enlace al nodo anterior y otro al siguiente. El beneficio no es decorativo: se gana navegación reversible sin reconstruir el camino desde la cabeza.

En un historial terapéutico esto significa comparar rápidamente una sesión reciente con una previa para identificar variaciones en el progreso.

Operaciones y costo conceptual

Joyanes Aguilar (2014) advierte que toda mejora funcional también trae un costo: más memoria por nodo y mayor cuidado en inserciones y eliminaciones. Al quitar un nodo, deben reajustarse dos enlaces y no uno solo. En términos pedagógicos, esto obliga a los estudiantes a pensar en coherencia estructural antes de ejecutar.

Navegación cronológica y comparativa

Weiss (2014) plantea que la elección de estructura debe responder a la operación frecuente. Si el terapeuta necesita avanzar cronológicamente y luego retroceder para contrastar el indice_progreso, la lista doblemente enlazada es más pertinente que una simple. En NeuroBiz el valor es decisional: cada sesión se vuelve un punto navegable del proceso clínico.

Variante circular doble

Cuando el flujo no tiene principio ni fin operativo —por ejemplo, un sistema de turnos continuos— la lista doblemente enlazada circular permite navegar indefinidamente en ambos sentidos. Esta variante será útil para pensar la HTI A, donde la continuidad del servicio exige retroceder y avanzar sin salir de la rotación.

🐍 Python como herramienta analítica de Business Intelligence

Python permite modelar el historial terapéutico como una secuencia navegable. La lectura clave está en cómo el docente transforma el recorrido en preguntas clínicas y de gestión.

Bloque 1 · NodoDoble y ListaDobleInserción, eliminación y recorridos en ambos sentidos

class NodoDoble:
    def __init__(self, dato):
        self.dato = dato
        self.anterior = None
        self.siguiente = None


class ListaDoble:
    def __init__(self):
        self.cabeza = None
        self.cola = None

    def insertar_final(self, dato):
        nuevo = NodoDoble(dato)
        if self.cabeza is None:
            self.cabeza = self.cola = nuevo
            return
        self.cola.siguiente = nuevo
        nuevo.anterior = self.cola
        self.cola = nuevo

    def eliminar(self, criterio):
        actual = self.cabeza
        while actual:
            if criterio(actual.dato):
                if actual.anterior:
                    actual.anterior.siguiente = actual.siguiente
                else:
                    self.cabeza = actual.siguiente
                if actual.siguiente:
                    actual.siguiente.anterior = actual.anterior
                else:
                    self.cola = actual.anterior
                return True
            actual = actual.siguiente
        return False

    def recorrer_adelante(self):
        datos = []
        actual = self.cabeza
        while actual:
            datos.append(actual.dato)
            actual = actual.siguiente
        return datos

    def recorrer_atras(self):
        datos = []
        actual = self.cola
        while actual:
            datos.append(actual.dato)
            actual = actual.anterior
        return datos


historial = ListaDoble()
for sesion in [
    {'fecha': '2026-04-03', 'indice_progreso': 0.52},
    {'fecha': '2026-04-10', 'indice_progreso': 0.57},
    {'fecha': '2026-04-17', 'indice_progreso': 0.60},
    {'fecha': '2026-04-24', 'indice_progreso': 0.66},
]:
    historial.insertar_final(sesion)

print('Adelante:', historial.recorrer_adelante())
print('Atrás:', historial.recorrer_atras())

def comparar_sesiones(lista_doble):
    actual = lista_doble.cabeza
    while actual and actual.siguiente:
        actual_progreso = actual.dato['indice_progreso']
        siguiente_progreso = actual.siguiente.dato['indice_progreso']
        delta = round(siguiente_progreso - actual_progreso, 2)
        print(
            f"{actual.dato['fecha']} -> {actual.siguiente.dato['fecha']}: cambio {delta:+}"
        )
        actual = actual.siguiente


comparar_sesiones(historial)
