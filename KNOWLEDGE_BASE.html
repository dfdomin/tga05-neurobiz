Sistema de Gamificación

Herramientas externas integradas

ClassDojo — gestión de puntos/comportamiento en clase (docente controla en tiempo real)
Quizizz — quizzes gamificados con ranking, tiempo y retroalimentación inmediata
Google Colab — ambiente Python en la nube (sin instalación)

⭐ Sistema XP (puntos de experiencia)

  • +5 XP al registrar nombre (primera visita)
  • XP por leer material semanal
  • XP por responder quizzes (proporcional al score)
  • XP por completar actividades (activity_done = true)
  • Barra de XP visible siempre en widget (color amarillo IUB)

🏅 Badges / niveles

  • Nivel basado en XP acumulado (Novato → Analista → Experto)
  • Mostrado en header de cada semana
  • Dashboard personal muestra histórico de badges

🧠 Taxonomía de Bloom en quizzes

  • Cada semana etiqueta sus preguntas con nivel Bloom
  • Recordar · Comprender · Aplicar · Analizar · Evaluar · Crear
  • Color diferenciado por nivel en la UI

💾 Widget de progreso (en cada semana)

  • Botón ☁️ Guardar (prominente) → Supabase
  • Botón ⚙️ → panel oculto con Código / Cargar (respaldo)
  • Modal bienvenida en primera visita (pide nombre + grupo)
  • Todos los datos en localStorage como respaldo local
⚠️
Conservar siempre: El componente de gamificación no debe modificarse al cambiar colores. El amarillo #FFDF2D IUB coincide exactamente con el rol del --gold de XP.

Arquitectura técnica

Patrón JAMstack — sin servidor

HTML estático en GitHub Pages + Supabase REST API. No requiere servidor propio. El alumno abre el link, el JS del HTML llama directamente a Supabase con la clave anon. RLS de Supabase protege los datos.

☁️ Supabase (base de datos)

  • Tabla: student_progress
  • UNIQUE(student_id, semana) → sin duplicados
  • RLS: anon puede INSERT, UPDATE, SELECT
  • View resumen_docente agrega XP, semanas, quizzes
  • Clave anon pública en GitHub es segura con RLS

💾 localStorage (respaldo local)

  • tga05_global — nombre, id, grupo
  • tga05_sN — estado semana N
  • tga05_supabase_url/key — config
  • tga05_visited_sN — historial visitas
  • tga05_parciales_corteN — notas parcial docente

🔄 Flujo de sincronización

Estudiante abre semana
  └→ Modal bienvenida si nombre vacío (+ 5 XP)
      └→ Completa material / quiz / actividad
          └→ Click ☁️ Guardar
              └→ syncCloud() → UPSERT a Supabase
                  └→ Fallback: Google Sheets si Supabase falla
                      └→ Siempre guarda en localStorage

// js/supabase-config.js
(function(){
  var SUPABASE_URL = "https://XXX.supabase.co";

Fórmula de calificación

Nota formativa (calculada automáticamente)

nota_formativa =
  (quiz_avg / 10 × 5.0) × 0.50  # Quiz promedio → 50%
  + (actividades / sem_esperadas × 5.0) × 0.30  # Actividades → 30%
  + (semanas_visitadas / sem_esperadas × 5.0) × 0.20  # Visitas → 20%

nota_corte = nota_formativa × 0.70 + nota_parcial × 0.30
# Pesos configurables en dashboard/notas.html

Diseño de Actividades Autocalificables

🎯 Principio rector

Toda actividad debe generar feedback inmediato y específico al estudiante, registrar el resultado en Supabase y contribuir a la nota formativa. El estudiante debe saber en el momento qué hizo bien, qué falló y por qué. La autocalificación no sustituye el criterio docente — lo libera para intervenciones de mayor valor.

✅ Feedback correcto

  • ✅ Ícono verde + explicación de por qué es correcto
  • +XP otorgado inmediatamente (barra animada)
  • Conexión explícita con el RA o criterio de evaluación
  • Invitación a profundizar (opcional)

❌ Feedback incorrecto

  • ❌ Sin penalización de XP — solo oportunidad perdida
  • Pista dirigida al concepto específico que falló
  • Referencia a sección del material para repasar
  • Posibilidad de reintentar (configurable por tipo)

🗂 Tipos de actividad por competencia

TipoDimensiónCuándo usarloMódulos
Opción múltiple con justificación Conceptual Definiciones, principios, comparaciones Todos
Arrastrar y soltar (clasificación) Conceptual Procedimental Ordenar pasos, clasificar elementos, relacionar columnas Todos
Completar código / blanks Procedimental Funciones con huecos, corregir errores de sintaxis Código (TGA05+)
Mini-editor de código + ejecución Procedimental Practicar funciones, estructuras de datos, algoritmos Código (TGA05+)
Escenario + decisión Actitudinal Conceptual Dilemas éticos, decisiones administrativas, análisis de caso Administración, gestión
Verdadero / Falso con argumento Conceptual Desmitificar ideas erróneas frecuentes Todos
Ordenar secuencia (timeline/proceso) Procedimental Fases de un proceso, pasos de un algoritmo Todos
Reflexión guiada (texto libre + keywords) Actitudinal No autocalificable en nota, pero genera reflexión y XP por completar Todos

💻 Módulos con código — estrategia específica

Opción A — Mini-editor embebido (JS)

  • Editor textarea o CodeMirror lite en el HTML
  • JS evalúa la salida contra resultado esperado
  • Feedback: "Tu función devuelve X, se esperaba Y"
  • Registra intento en quiz_answers JSONB

Opción B — Google Colab (Python)

  • Notebook estructurado: teoría → ejercicio → celda de prueba
  • Celda de validación con assert + mensaje amigable
  • Estudiante completa función, ejecuta y ve ✅ o ❌
  • Guarda evidencia en Drive; docente puede revisar
💡
Para TGA05 y módulos similares: usar Opción B (Colab) como actividad principal de código + mini-editor en HTML para ejercicios de calentamiento rápido (5–10 min).

🔍 Crítica de contenido — transparencia curricular

Cuando el microcurrículo incluye temas con alto volumen, redundancia o desfase tecnológico, el material debe ser honesto con el estudiante. Esto no resta profundidad — aumenta la confianza y la motivación.

EtiquetaSignificadoAcción en el material
🟢 VIGENTE Tema actual, ampliamente usado Presentar normalmente, sin nota adicional
🟡 EVOLUCIONADO El concepto aplica, pero la herramienta/sintaxis cambió Enseñar el concepto + nota "Hoy se usa X en su lugar"
🔴 HISTÓRICO Obsoleto en producción moderna Explicar brevemente + decir qué lo reemplaza + cuándo aún aplica

📝 Plantilla de nota crítica (insertar en el material)

⚠️ Nota de contexto:
Este tema (ej: "Bubble Sort") forma parte del currículo oficial.
Lo estudiamos para entender [principio fundamental: comparación, complejidad].
En la práctica profesional actual ha sido reemplazado por [QuickSort / algoritmos
de librería estándar], que son más eficientes. Sin embargo, Bubble Sort sigue
siendo útil para: [docencia, datasets muy pequeños, demostraciones paso a paso].
Enfócate en comprender el concepto, no en memorizarlo para producción.

Rutina docente de revisión (perfil neurodivergente)

🧠 Diseñado para ASD nivel 1 + TDAH + AACC

El objetivo es que revisar el trabajo de los estudiantes sea predecible, breve y recompensante. Sin listas infinitas, sin ambigüedad, con inicio y fin claros. La herramienta está en dashboard/revision.html.

⏱ Estructura de sesión (15 min exactos)

  1. Abrir dashboard/revision.html
  2. Clic ▶ Iniciar — el timer arranca
  3. Revisar estudiantes en riesgo primero (tab rojo)
  4. Un estudiante a la vez — expandir → revisar → marcar ✅
  5. Al terminar → celebración automática + XP ganado
  6. Cerrar la pestaña — done.

📅 Cuándo hacerlo (ritmo semanal)

  • Martes post-clase (15 min) — revisar HTI-A de la semana anterior mientras el contexto está fresco
  • Viernes (15 min) — revisar estudiantes en riesgo antes del fin de semana
  • Total: 30 min/semana máximo
  • Si hay poco tiempo: solo el tab ⚠️ En riesgo. El sistema guarda quién ya fue revisado.

🎮 Gamificación del docente — sistema XP

Por cada revisión: +10 XP

Por sesión completa: +20 XP bonus

XP acumulado guardado en localStorage → historial de sesiones.

Al terminar todos los pendientes: pantalla de celebración animada

El estado se resetea cada día — cada mañana es un nuevo comienzo sin el peso acumulado.

🔑 Principios de diseño para este perfil

NecesidadSolución en la herramienta
ASD: predictibilidad, estructura claraMismo flujo siempre: abrir → timer → revisar por prioridad → cerrar
TDAH: foco, sesiones cortas, recompensaTimer de 15 min visible, XP inmediato por cada acción, celebración al final
AACC: eficiencia, no repetir lo innecesarioOrden automático por riesgo, notas rápidas por estudiante, sin formularios largos
Evitar acumulación de deuda cognitivaReset diario automático — lo no revisado hoy espera mañana sin culpa
Contexto siempre visibleSemanas visitadas, quizzes, XP y fecha de última actividad en una sola pantalla
💡
Para replicar en otro módulo: dashboard/revision.html es reutilizable sin cambios. Solo necesita supabase-config.js configurado.

Importar lista de estudiantes desde PDF Academusoft

🎯 Por qué existe esta función

El módulo se reutiliza cada cuatrimestre y con grupos distintos. En lugar de editar el código fuente para cambiar la lista de estudiantes, el sistema detecta automáticamente la lista a partir del reporte PDF estándar que genera Academusoft (IUB). Una vez importado, el listado queda guardado en localStorage y persiste entre sesiones.

📋 Dónde está la función

  • Archivo: dashboard/participacion.html
  • Acceso: Tab ⚙️ Configuración → tarjeta "📋 Importar lista de estudiantes"
  • PIN de docente requerido para acceder

🗄️ Storage keys

Key localStorageContenido
tga05_students_v1Array JSON con los estudiantes importados
tga05_group_info_v1Metadata del grupo: grupo, horario, salón, asignatura

⚙️ Dos formas de importar

OpciónPasosCuándo usar
A — PDF directo Configuración → subir PDF → esperar extracción automática con PDF.js Cuando tienes el archivo PDF del reporte descargado
B — Pegar texto Abrir PDF en cualquier visor → Ctrl+A → Ctrl+C → pegar en el campo → Analizar Cuando el PDF está en el navegador o PDF.js falla por protección del archivo

📄 Formato del reporte Academusoft esperado

El parser detecta automáticamente líneas con este patrón:

# Número + nombre en mayúsculas + cédula (7-12 dígitos)
1    ARIZA DE LA CRUZ YORYELIS PATRICIA      1043158867
2    BERRIO JULIO BLEIDYS                    1001234567
...
# También detecta metadatos del encabezado:
GRUPO: 7_GA_G1
HORARIO: Viernes 07:00 - 10:59
SALÓN: CENH P1-7
ASIGNATURA: ESTRUCTURA DE DATOS PARA LOS NEGOCIOS
⚠️
Si el PDF extrae texto con columnas desordenadas (nombre y cédula en líneas separadas), el parser intentará un modo de recuperación más amplio. En caso de fallo, el estudiante aparecerá con CC = SIN_DOC_N — esto es editable manualmente en el código.

🔁 Flujo completo para nuevo cuatrimestre / nuevo grupo

  1. Descargar el reporte de estudiantes desde Academusoft (PDF)
  2. Abrir dashboard/participacion.html → ingresar PIN
  3. Ir a tab ⚙️ Configuración → tarjeta Importar lista
  4. Subir el PDF (Opción A) o pegar texto (Opción B)
  5. Revisar la vista previa — verificar nombres y cédulas
  6. Clic ✅ Cargar N estudiantes
  7. El encabezado se actualiza automáticamente con el nombre del grupo
  8. Listo — la lista persiste en este dispositivo para todas las clases del cuatrimestre
💡
Los datos de participación NO se borran al cambiar de lista. Si empiezas un nuevo cuatrimestre, borra primero los eventos desde Zona de peligro → "Borrar todos los datos de participación".

🛠️ Para replicar en otro módulo

El importador es completamente portable. Para usarlo en otro módulo:

  1. Copiar dashboard/participacion.html al nuevo módulo
  2. Cambiar el prefijo del storage key (tga05_) por el código del nuevo módulo (ej: tga03_) para evitar conflictos entre módulos en el mismo navegador
  3. La lista por defecto (hardcoded) puede quedar vacía o con estudiantes de ejemplo — siempre se puede sobrescribir por importación
  4. Asegurarse de que el nuevo módulo tenga el mismo STORE_KEY en dashboard/notas.html
✅
El sistema fue diseñado exactamente para este uso recurrente: un módulo diferente cada cuatrimestre, grupos distintos, sin tocar código. El importador es la pieza clave que lo hace posible.
