1. ¿Qué es una lista enlazada simple?

Una lista enlazada simple es una estructura dinámica compuesta por nodos. Cada nodo almacena un dato y una referencia al siguiente nodo. Joyanes Aguilar (2014) la presenta como una alternativa flexible frente a los arreglos cuando el tamaño de la colección puede cambiar durante la ejecución.

Lo importante para esta semana no es memorizar tecnicismos de memoria, sino comprender la metáfora: cada sesión del historial no necesita “saberlo todo”; solo necesita guardar su información y apuntar a la siguiente sesión.

2. Nodo = dato + puntero/referencia

Cairó Battistutti y Guardati Bueno (2006) explican que el nodo es la unidad mínima de la estructura enlazada. En Python solemos representarlo con una clase que guarda el dato y un atributo siguiente. Aunque Python abstrae detalles de memoria, la idea conceptual sigue siendo la misma.

Weiss (2014) subraya que comprender la referencia entre nodos es esencial para dominar operaciones posteriores. Sin esa imagen mental, insertar o eliminar nodos se vuelve una secuencia mecánica sin sentido estructural.

3. Diferencia con arreglos

En un arreglo, los elementos ocupan posiciones consecutivas y el acceso por índice es muy cómodo. Sin embargo, insertar un elemento “en medio” puede requerir desplazar varios registros. En una lista enlazada, el costo conceptual se mueve: acceder por posición es menos directo, pero insertar nodos puede ser más natural si ya llegaste al punto correcto.

Para un historial terapéutico que se corrige o crece durante el tiempo, esta flexibilidad tiene valor administrativo. La estructura ya no se piensa como un bloque fijo, sino como una cadena viva de eventos.

4. Crear y recorrer una lista enlazada en Python

Python se usará como herramienta analítica para construir el modelo. La operación mínima de la semana consiste en crear nodos, enlazarlos y recorrer la estructura para producir una salida legible. Ese recorrido es la base para búsquedas, conteos y futuras operaciones CRUD.

Moreno Pérez (2012) recomienda reforzar estas estructuras con diagramas antes del código. Por eso el trabajo presencial combina dibujo, pseudocódigo y ejecución en Colab.

Bloque 2 — Lista enlazada con inserción al inicio y al final

Este bloque crea la estructura base que se usará en clase para el historial terapéutico.

Google Colab · Bloque 2Abrir Colab
class ListaEnlazada:
    def __init__(self):
        self.cabeza = None

    def insertar_al_inicio(self, dato):
        nuevo = Nodo(dato)
        nuevo.siguiente = self.cabeza
        self.cabeza = nuevo

    def insertar_al_final(self, dato):
        nuevo = Nodo(dato)
        if self.cabeza is None:
            self.cabeza = nuevo
            return
        actual = self.cabeza
        while actual.siguiente is not None:
            actual = actual.siguiente
        actual.siguiente = nuevo

    def recorrer(self):
        actual = self.cabeza
        while actual:
            print(actual.dato)
            actual = actual.siguiente

Sugerencia: ejecuta el bloque, cambia un dato del caso NeuroBiz y registra qué ocurre. Esa modificación convierte el ejemplo en evidencia de comprensión.
