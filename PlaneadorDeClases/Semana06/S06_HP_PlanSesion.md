## 3. Protocolo de predictibilidad ACCC
- Iniciar nombrando exactamente qué cambia respecto al HAD: ahora cambia el recurso, no el objetivo general.
- Mantener la misma tabla de datos NeuroBiz visible en la parte superior o en una ventana lateral.

## 5. Quizizz/Wayground: configuración y preguntas sugeridas (mínimo 6)
**Configuración sugerida**
- Modo: Playground.
- Activación: primer descanso de voz del bloque HP.
- Cantidad: 6 preguntas.
- Tiempo: 20-25 segundos.
- Retroalimentación: inmediata.
- Orden aleatorio: activado.
- Código listo antes del inicio del laboratorio.
- Si no conecta: usar lectura de pantallas y preguntas orales rápidas.

**Preguntas sugeridas**
1. En Python, la palabra reservada de la semana es: `if` / `while` / `import` / `class`.  
2. Después de `if n4 > 15:` debe venir: una línea indentada / `FinSi` / una tabla / `case`.  
3. Si `n4 = 30.56`, el programa de alerta debe mostrar mensaje para: T06 / T08 / T04 / T01.  
4. En PSeInt, la estructura se cierra con: `FinSi` / `FinMientras` / `Hasta Que` / `Segun`.  
5. Si la condición es falsa, en esta semana el programa: continúa sin ejecutar el bloque / repite / cambia a else / termina con error.  
6. Thonny se usa en el curso para: escribir y ejecutar Python / dibujar flujos / navegar web / hacer diapositivas.  

**Clave:** 1 `if`, 2 una línea indentada, 3 T06, 4 `FinSi`, 5 continúa sin ejecutar el bloque, 6 escribir y ejecutar Python.

## 7. Nota de conexión neurodivergencia (1-2 párrafos)
El enfoque práctico favorece a estudiantes con **TDAH** porque hace visible la secuencia exacta: leer un dato, evaluar una condición y mostrar un mensaje. La estructura es corta, concreta y comprobable. El ejemplo de T06 funciona bien porque conecta un criterio real de seguimiento clínico con una decisión binaria comprensible: hay alerta o no hay alerta.

Además, el laboratorio reduce carga verbal continua. El docente explica poco tiempo seguido y luego monitorea. Esto protege la voz del profesor, disminuye saturación auditiva y deja más espacio para que cada estudiante procese a su ritmo en PSeInt y Thonny.

## 8. Contingencias (tabla: Situación | Ruta A | Ruta B)
| Situación | Ruta A | Ruta B |
|---|---|---|
| Thonny no abre en algunos equipos | Trabajar primero en PSeInt y luego usar un equipo monitor por pareja para Python. | Pasar a transcripción en cuaderno del código Python y ejecutar solo en el equipo del docente. |
| PSeInt falla | Hacer el algoritmo en texto plano respetando la sintaxis visible. | Ir directo a Python y mantener el pseudocódigo escrito en cuaderno. |
| Proyector falla | El docente dicta la plantilla mínima y la escribe en tablero. | Usar captura impresa o compartir el archivo por grupo de mensajería institucional si está autorizado. |
| Quizizz no conecta | Convertirlo en preguntas orales de sintaxis y otorgar puntos Dojo. | Reemplazar por revisión entre parejas con lista de chequeo. |
| Retraso de estudiantes después del descanso | Mantener demostración guiada y mover el Quizizz al final como salida. | Reducir el reto adicional y conservar solo programa base + evidencia. |
