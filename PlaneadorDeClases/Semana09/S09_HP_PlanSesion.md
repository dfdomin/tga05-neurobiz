## 3. Protocolo predictibilidad ACCC
- Mostrar la agenda del laboratorio desde antes de que el grupo se siente.
- La voz del docente se organiza en bloques cortos: explica, modela una línea, calla y monitorea.
- Frase de transición fija: **“Cerramos bloque, abrimos bloque, cambia solo la herramienta.”**
- Si un computador falla, no se improvisa: se activa la contingencia escrita y la meta no cambia.
- Todo ejemplo de código debe cerrar con una salida esperada clara.

## 5. Quizizz/Wayground (6-8 preguntas sugeridas)
1. `for i in range(1, 6)` imprime: **1 a 5** / 1 a 6 / solo 6.  
2. Para promediar satisfacción de 8 terapeutas conviene `for` porque la cantidad es: **conocida** / infinita / secreta.  
3. La satisfacción promedio del equipo es: **4.61** / 4.20 / 5.00 / 16.7.  
4. HH-FHH ejecuta el cuerpo: **antes de evaluar la condición** / solo si la condición ya es verdadera.  
5. U1 se calcula con: **142/150 * 100** / 12/150 * 100 / 2/12 * 100.  
6. U3 correcto es: **16.7%** / 94.7% / 88.4% / 75%.  
7. Python simula HH-FHH con: **while True y break** / else forever / pass.  
8. Un uso real de `for` en NeuroBiz es: **N sesiones predefinidas** / ciclo sin fin / contraseña incorrecta.

## 7. Nota neurodivergencia
En esta semana la mención a **TEA** debe centrarse en la importancia de la **rutina, estructura y predictibilidad**, nunca en estereotipos. El `for` es útil como metáfora porque define desde el inicio cuántas repeticiones habrá. El docente puede decir: “En algunos procesos terapéuticos NeuroBiz programa exactamente N sesiones; esa previsibilidad se parece a un ciclo contador”.
