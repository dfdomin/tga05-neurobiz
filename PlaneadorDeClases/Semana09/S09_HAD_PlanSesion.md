## 5. Quizizz/Wayground (6-8 preguntas sugeridas)
1. Un `for` se usa mejor cuando el número de repeticiones es **conocido** / infinito / secreto.  
2. En PSeInt, PARA-FPARA necesita: **inicio, fin e incremento** / solo una variable / solo imprimir.  
3. HH-FHH garantiza ejecutar el bloque: **al menos una vez** / cero veces / exactamente dos veces.  
4. El promedio de satisfacción de T01–T08 es: **4.61** / 3.90 / 5.00 / 16.7.  
5. `range(1, 6)` en Python produce: **1,2,3,4,5** / 1 a 6 incluyendo 6 / solo 6.  
6. U1 del contrato UIB es: **94.7%** / 16.7% / 75% / 88.4%.  
7. Un ejemplo cercano a `for` por predictibilidad es: **N sesiones predefinidas para TEA** / un ciclo infinito / una variable lógica.  
8. Python no trae `do-while` nativo; se simula con: **while True + break** / import do / float loop.

## 7. Nota neurodivergencia
**TEA.** Esta semana la idea de repetición se vincula con la **predictibilidad**. Para muchas personas con TEA, las rutinas conocidas y el número definido de pasos reducen incertidumbre. Eso se parece a un `for`: se sabe cuántas iteraciones habrá, en qué orden y con qué estructura. El docente puede mencionar que NeuroBiz diseña terapias con exactamente **N sesiones predefinidas** para algunos procesos, y esa organización se conecta mejor con PARA-FPARA que con un ciclo abierto.
