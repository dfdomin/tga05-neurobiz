## 3. Protocolo predictibilidad ACCC
- El bloque inicia con la agenda exacta ya proyectada; nadie adivina “qué sigue”.
- Se mantiene la regla de voz: máximo 20-25 min continuos en **HABLA**.
- La transición de recurso se dice siempre igual: **“Cerramos bloque, abrimos bloque, cambia solo la herramienta.”**
- Toda contingencia debe estar escrita antes de iniciar: internet, equipos lentos, errores de sintaxis, cortes de energía.
- En práctica guiada, el docente habla corto, muestra una línea, se calla y monitorea ejecución real.

## 5. Quizizz/Wayground (6-8 preguntas sugeridas)
1. La línea que casi siempre evita un ciclo infinito es la **actualización** / impresión / comentario.  
2. `contador = contador + 1` debe ir: **dentro del while** / fuera del programa / antes del `import`.  
3. La suma correcta de realizadas T01–T08 es: **520** / 57 / 142 / 150.  
4. El promedio N1 del equipo es: **88.4%** / 59.7% / 94.7% / 4.61.  
5. `while indice < len(lista):` usa una variable de control llamada: **indice** / print / input / break.  
6. Un `while` puede no ejecutarse ninguna vez si la condición inicial es: **falsa** / verdadera / decimal / larga.  
7. T06 tiene N1 y N4 en riesgo: **sí** / no.  
8. En Thonny, para ejecutar rápido se usa: **Run/F5** / Ctrl+P / Tab / Insert.

## 7. Nota neurodivergencia
La semana 8 usa el caso **dislexia** para que la repetición tenga sentido humano. No se presenta como “repetir por repetir”, sino como sostener un proceso: mientras haga falta apoyo lector, el acompañamiento continúa. El laboratorio debe mostrar que el ciclo necesita una condición de salida clara, igual que en intervención real se necesitan criterios de avance y seguimiento. El ejemplo de **T03** ayuda a hablar de interrupciones del proceso por las **14 canceladas**.
