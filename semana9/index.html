1. Operar una estructura es comprenderla de verdad

Construir la lista enlazada fue el primer paso; ahora la comprensión se evidencia cuando el estudiante manipula referencias sin perder la integridad de la estructura. Weiss (2014) insiste en que las operaciones revelan el verdadero costo y valor de una estructura de datos.

En términos pedagógicos, pasar de crear a operar implica subir en Bloom: el estudiante ya no solo reconoce componentes, sino que aplica procedimientos y analiza sus efectos sobre el sistema completo.

2. Insertar, eliminar y buscar

Insertar en posición exige recorrer hasta el punto correcto y luego reencadenar enlaces. El concepto clave es que una operación local puede afectar la coherencia global si no se actualizan bien las referencias.

Eliminar nodos obliga a pensar en el nodo actual, el anterior y los casos especiales: lista vacía, cabeza y elemento inexistente. Joyanes Aguilar (2014) subraya que los algoritmos sobre listas deben contemplar explícitamente estos escenarios límite.

3. Invertir una lista: leer el historial desde el presente

Invertir una lista enlazada simple es una operación poderosa porque obliga a reconsiderar la dirección de cada enlace. El caso NeuroBiz la justifica cuando la coordinación desea ver primero las sesiones más recientes.

Cairó Battistutti y Guardati Bueno (2006) ayudan a comprender que invertir no es “copiar al revés”, sino reconstruir la dirección de las referencias. Esa distinción fortalece la comprensión estructural.

4. CRUD y trazabilidad en un contexto de negocio

Aunque aquí no trabajamos con una base de datos formal, las operaciones se acercan a una lógica CRUD: crear nodos, leer/recorrer, actualizar mediante inserción y eliminar registros cancelados. Esta conexión ayuda a que el estudiante vea continuidad con sistemas administrativos reales.

Moreno Pérez (2012) recomienda verificar cada operación con ejemplos pequeños antes de escalar. Por eso en clase se prueban listas de 3 a 5 nodos antes de pasar a historiales más ricos.

Bloque 1 — Buscar y eliminar sesiones canceladas

Primera capa de operaciones sobre la lista enlazada del historial.

Google Colab · Bloque 1Abrir Colab
class Nodo:
    def __init__(self, dato):
        self.dato = dato
        self.siguiente = None

class ListaEnlazada:
    def __init__(self):
        self.cabeza = None

    def insertar_al_final(self, dato):
        nuevo = Nodo(dato)
        if self.cabeza is None:
            self.cabeza = nuevo
            return
        actual = self.cabeza
        while actual.siguiente:
            actual = actual.siguiente
        actual.siguiente = nuevo

    def buscar(self, id_sesion):
        actual = self.cabeza
        while actual:
            if actual.dato["id"] == id_sesion:
                return actual.dato
            actual = actual.siguiente
        return None

    def eliminar(self, id_sesion):
        actual = self.cabeza
        anterior = None
        while actual:
            if actual.dato["id"] == id_sesion:
                if anterior is None:
                    self.cabeza = actual.siguiente
                else:
                    anterior.siguiente = actual.siguiente
                return True
            anterior = actual
            actual = actual.siguiente
        return False

print("\n✓ Pipeline completado — historial listo para auditoría")
