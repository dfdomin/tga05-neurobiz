1. Recursividad: una definición útil para negocios

Cairó Battistutti y Guardati Bueno (2006) presentan la recursividad como una técnica en la cual un problema se resuelve descomponiéndolo en una versión más pequeña del mismo problema. En clave administrativa, esto significa que una política de cálculo puede repetirse mientras exista un “resto” de información pendiente.

Joyanes Aguilar (2014) insiste en que la definición recursiva exige dos piezas inseparables: un caso base que detiene el proceso y un caso recursivo que aproxima la solución al caso base. Si una de las dos piezas falta, el algoritmo deja de ser confiable.

2. Caso base y caso recursivo en NeuroBiz

Cuando NeuroBiz calcula el progreso acumulado de un paciente, el caso base puede definirse como “si no hay sesiones pendientes, el acumulado es 0”. El caso recursivo, en cambio, toma el progreso de la sesión actual y le suma el acumulado obtenido hasta la sesión anterior.

Weiss (2014) recuerda que la corrección de un algoritmo recursivo no depende solo de que “parezca elegante”, sino de demostrar que cada llamada reduce el tamaño del problema. En este módulo, esa reducción se observa cuando N pasa de 6 a 5, luego a 4, hasta llegar a 0.

3. Árbol de llamadas y pila conceptual

El árbol de llamadas permite visualizar qué invocaciones nacen de una función. Para estudiantes que recién ingresan al tema, este recurso es clave porque transforma una idea abstracta en una secuencia trazable. Moreno Pérez (2012) sugiere usar trazas y tablas de seguimiento para comprender cómo el flujo vuelve “hacia arriba” una vez se alcanza el caso base.

En Python, cada llamada queda temporalmente en la pila de ejecución. No se profundiza aquí en memoria técnica avanzada; basta comprender que cada nivel “espera” la respuesta del siguiente. Esa espera es exactamente la razón por la que conviene escribir funciones pequeñas, claras y bien delimitadas.

4. Ejemplos clásicos y ejemplo de negocio

El factorial y Fibonacci aparecen con frecuencia porque muestran patrones sencillos de repetición. El factorial ilustra una recursividad lineal fácil de seguir; Fibonacci, en cambio, exhibe rápidamente el costo de repetir cálculos, lo que permite anticipar conversaciones posteriores sobre eficiencia.

Trasladado a NeuroBiz, el ejemplo central es más significativo: progreso_total(N) = progreso_sesion_N + progreso_total(N-1). Aquí Python funciona como herramienta de inteligencia de negocios: ayuda a explicar y automatizar un indicador, no a convertir la clase en un curso exclusivo de sintaxis.

Bloque 1 — Factorial recursivo y visualización del árbol de llamadas

Úsalo para recordar la estructura mínima de una función recursiva y para explicar el rol del caso base.

Google Colab · Bloque 1Abrir Colab
# Ejemplo clásico para comprender la estructura recursiva
def factorial(n, nivel=0):
    sangria = "  " * nivel
    print(sangria + f"factorial({n})")
    if n == 0 or n == 1:
        print(sangria + "↳ caso base = 1")
        return 1
    resultado = n * factorial(n - 1, nivel + 1)
    print(sangria + f"↩ retorna {resultado}")
    return resultado

print("Factorial de 5:", factorial(5))

Bloque 2 — Índice de progreso acumulado de un paciente NeuroBiz

Este bloque modela el caso transversal real del módulo: sumar el progreso de cada sesión desde la última hasta llegar al caso base.

Google Colab · Bloque 2Abrir Colab
# Cada valor representa el indice_progreso alcanzado en una sesión
progresos = [0.8, 1.1, 0.9, 1.3, 0.7]

def progreso_acumulado(lista, n=None):
    if n is None:
        n = len(lista)
    if n == 0:
        return 0
    return lista[n - 1] + progreso_acumulado(lista, n - 1)

total = progreso_acumulado(progresos)
print(f"Progreso acumulado del paciente: {total:.2f}")

Bloque 3 — Fibonacci como advertencia sobre repetición de cálculos

Incluye un contador para que el grupo observe por qué no toda función recursiva es automáticamente eficiente.

Google Colab · Bloque 3Abrir Colab
contador = {"llamadas": 0}

def fibonacci(n):
    contador["llamadas"] += 1
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

n = 6
valor = fibonacci(n)
print(f"fibonacci({n}) = {valor}")
print(f"Llamadas realizadas: {contador['llamadas']}")
