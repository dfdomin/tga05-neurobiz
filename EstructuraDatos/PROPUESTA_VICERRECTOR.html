El problema actual

😓 Proceso manual hoy

  • Docente lleva registro en Excel propio
  • Al cierre del corte: exporta, consolida, corrige errores
  • Ingresa nota única manualmente a Academusoft
  • Riesgo de error humano en transcripción
  • Sin visibilidad del progreso en tiempo real

📉 Costos ocultos

  • ~2–4 horas por docente por corte en solo el registro
  • Inconsistencias entre Moodle y Academusoft frecuentes
  • Estudiantes sin retroalimentación oportuna
  • Coordinadores sin datos agregados para intervención temprana
  • Procrastinación por alta carga cognitiva en tarea repetitiva

🔢 Escala del problema en IUB

Si un docente dicta 2 cursos de 30 estudiantes con 3 cortes/semestre, se trata de 180 notas manuales por docente al año solo en transcripción — sin contar la consolidación previa.

2

La solución propuesta

🎯 Visión: Un solo clic del docente para cerrar el corte

El sistema consolida automáticamente la nota formativa desde las actividades de Moodle. El docente solo ingresa la nota del parcial presencial. El sistema calcula la nota final y la envía a Academusoft.

✅ Lo que YA existe (piloto TGA05)

  • Material en GitHub Pages con tracking automático
  • Supabase guarda progreso de cada estudiante en tiempo real
  • Dashboard docente con notas formativas calculadas
  • Exportación a XLSX con fórmulas automáticas
  • Columna "Nota Parcial" para completar en 5 minutos

🔧 Lo que se necesita agregar

  • Conexión API Moodle → Supabase (o directo a Academusoft)
  • Formato de importación compatble con Academusoft
  • Validación y flujo de aprobación de notas
  • Autenticación institucional (SSO con credenciales IUB)

Hoja de ruta de implementación

Fase 1 — Piloto activo EN CURSO
TGA05 con GitHub Pages + Supabase + Dashboard docente. Consolidación semi-automática: formativa calculada, parcial manual, exportación XLSX.
Fase 2 — Integración Moodle Grades API 2–4 semanas
Las actividades de Moodle alimentan Supabase automáticamente. El docente ve notas formativas actualizadas sin intervención. Requiere habilitar Web Services en Moodle de IUB.
Fase 3 — Formato de importación Academusoft 1–2 semanas
Generar automáticamente el archivo (CSV/Excel) en el formato exacto que Academusoft acepta para importación masiva. El docente solo descarga y sube.
Fase 4 — Conexión directa Supabase → Academusoft Requiere cooperación TI
Si Academusoft expone una API REST o permite integración, el sistema enviará las notas automáticamente al cierre del corte. Elimina por completo la intervención manual.
Fase 5 — Escalado a todos los módulos TGA 1 sprint de desarrollo
Replicar la arquitectura a los demás módulos del programa. El material HTML se genera con IA en horas. La infraestructura ya está probada.

Requerimientos para escalar

📋 De Vicerrectoría Académica

  • Socialización con docentes interesados

💡 Costo de la solución

Infraestructura: $0 — Supabase (plan gratuito cubre hasta 50.000 filas), GitHub Pages (gratuito para repos públicos y organizaciones educativas).


Desarrollo: Ya existe el 70% del sistema. Las fases 2 y 3 son estimadas en 3–6 semanas de trabajo con apoyo de IA (como se está haciendo en este piloto).
