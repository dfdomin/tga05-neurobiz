Teoría aplicada a negocios

La teoría se construye con base en literatura clásica de estructuras de datos y se traduce al lenguaje operativo de NeuroBiz.

1. De la pila como contenedor a la pila como intérprete

La semana anterior la pila se entendió como estructura para acceder al dato más reciente. Ahora el mismo principio se aplica a problemas simbólicos. Weiss (2014) muestra que muchas aplicaciones clásicas de las pilas incluyen la evaluación de expresiones, el manejo de llamadas y la verificación de estructuras anidadas. Esto es valioso en negocios porque las fórmulas deben interpretarse de forma no ambigua.

Cuando una empresa escribe una fórmula como ( sesiones_asistidas / ( sesiones_asistidas + sesiones_fallidas ) ) * 100, la persona puede entenderla visualmente, pero el sistema necesita un procedimiento ordenado para procesar símbolos, operadores y paréntesis. La pila se convierte entonces en una herramienta de análisis, no solo de almacenamiento.

2. Infija, prefija y postfija: tres formas de representar una misma lógica

Joyanes Aguilar (2014) y Cairó Battistutti y Guardati Bueno (2006) explican que una expresión puede representarse de distintas maneras según dónde se ubique el operador. En notación infija, el operador aparece entre operandos; en prefija, antes; en postfija, después. La postfija resulta especialmente útil porque reduce la necesidad de paréntesis durante la evaluación.

NotaciónEjemplo con KPIObservación
Infija( a / ( a + f ) ) * 100Es la más familiar para humanos.
Prefija* / a + a f 100Coloca primero el operador; se usa menos en el aula inicial.
Postfijaa a f + / 100 *Facilita la evaluación con una pila.

En el caso NeuroBiz, la notación postfija puede verse como una representación interna del cálculo. El estudiante no necesita memorizar todos los algoritmos formales de compiladores; sí necesita comprender que una reorganización de símbolos permite evaluar paso a paso con reglas simples.

3. Evaluación de expresiones postfijas con pila

La lógica básica es directa: al leer un operando, se apila; al leer un operador, se desapilan los dos últimos operandos, se aplica la operación y el resultado se vuelve a apilar. Weiss (2014) destaca que esta estrategia elimina ambigüedades sobre precedencia, porque la estructura misma del recorrido define cuándo operar.

Desde Bloom, el paso importante esta semana es que el estudiante no solo aplique una secuencia mecánica, sino que analice cómo cambia la pila en cada token. Por eso conviene registrar el estado intermedio, no solo el resultado final.

4. Paréntesis balanceados y validación de fórmulas

Cairó Battistutti y Guardati Bueno (2006) usan las pilas para verificar símbolos balanceados. Si en una fórmula se abre un paréntesis y luego no se cierra, la pila conserva ese símbolo pendiente. Si aparece un cierre sin apertura previa, el sistema detecta inconsistencia. En términos organizacionales, esto equivale a validar que una fórmula esté bien construida antes de calcularla.

En NeuroBiz, este control evita que un analista ejecute indicadores con expresiones incompletas o mal digitadas. El valor de negocio es claro: una validación simple previene conclusiones incorrectas sobre retención, eficiencia o progreso terapéutico.

Idea clave: la pila no “entiende” el significado clínico del KPI, pero sí permite verificar si la fórmula está estructurada correctamente para ser interpretada por el sistema.

5. Traducción pedagógica del análisis

Moreno Pérez (2012) recuerda que programar es formalizar una solución previamente comprendida. Por eso, en esta semana interesa que el estudiante verbalice qué está ocurriendo cuando lee cada token: “apilo”, “desapilo”, “combino”, “vuelvo a apilar”. Ese lenguaje operativo ayuda a pasar del hacer al analizar.

Una expresión bien evaluada no es solo una cuenta correcta; es una secuencia de decisiones explícitas sobre cuándo almacenar temporalmente y cuándo operar. Esa secuencia es precisamente el aporte de la pila.

Bloque 1: Evaluador de expresión postfija para KPI de eficiencia colab_postfix_eval.py 
OPERADORES = {
    "+": lambda a, b: a + b,
    "-": lambda a, b: a - b,
    "*": lambda a, b: a * b,
    "/": lambda a, b: a / b,
}

def es_numero(token):
    try:
        float(token)
        return True
    except ValueError:
        return False

def evaluar_postfija(tokens, variables):
    pila = []

    for token in tokens:
        if token in OPERADORES:
            b = pila.pop()
            a = pila.pop()
            resultado = OPERADORES[token](a, b)
            pila.append(resultado)
            print(f"Operador {token}: pila -> {pila}")
        else:
            valor = float(token) if es_numero(token) else float(variables[token])
            pila.append(valor)
            print(f"Operando {token}: pila -> {pila}")

    if len(pila) != 1:
        raise ValueError("La expresión no quedó reducida a un único resultado")
    return pila[0]

variables = {
    "sesiones_asistidas": 196,
    "sesiones_fallidas": 12,
}

formula = "sesiones_asistidas sesiones_asistidas sesiones_fallidas + / 100 *".split()
eficiencia = evaluar_postfija(formula, variables)
print(f"Eficiencia NeuroBiz: {eficiencia:.2f}%")
Uso pedagógico: Permite calcular un KPI real usando una pila y deja visible el estado intermedio durante la lectura de tokens.
Bloque 2: Verificador de paréntesis balanceados en fórmulas colab_balance_checker.py 
def parentesis_balanceados(expresion):
    pila = []
    pares = {")": "(", "]": "[", "}": "{"}

    for simbolo in expresion:
        if simbolo in "([{":
            pila.append(simbolo)
        elif simbolo in ")]}":
            if not pila or pila[-1] != pares[simbolo]:
                return False
            pila.pop()
    return len(pila) == 0

formulas = [
    "( sesiones_asistidas / ( sesiones_asistidas + sesiones_fallidas ) ) * 100",
    "( indice_progreso + nivel_hiperfoco ] / 2",
    "(( reconocimiento_patrones + indice_progreso ) / 2)",
]

for formula in formulas:
    estado = parentesis_balanceados(formula)
    print(f"{formula} -> {'válida' if estado else 'inválida'}")
Uso pedagógico: Convierte la pila en un filtro de calidad previo al cálculo de indicadores.
Bloque 3: Conversión infija a postfija con tokens separados por espacios colab_infix_to_postfix.py 
PRECEDENCIA = {
    "+": 1,
    "-": 1,
    "*": 2,
    "/": 2,
}

def infija_a_postfija(expresion):
    salida = []
    operadores = []

    for token in expresion.split():
        if token.replace("_", "").isalnum():
            salida.append(token)
        elif token == "(":
            operadores.append(token)
        elif token == ")":
            while operadores and operadores[-1] != "(":
                salida.append(operadores.pop())
            operadores.pop()
        else:
            while (
                operadores
                and operadores[-1] != "("
                and PRECEDENCIA[operadores[-1]] >= PRECEDENCIA[token]
            ):
                salida.append(operadores.pop())
            operadores.append(token)

    while operadores:
        salida.append(operadores.pop())
    return salida

expresion = "( sesiones_asistidas / ( sesiones_asistidas + sesiones_fallidas ) ) * 100"
postfija = infija_a_postfija(expresion)
print("Infija:", expresion)
print("Postfija:", " ".join(postfija))
Uso pedagógico: Muestra el puente entre la expresión que entiende la persona y la secuencia que resulta más fácil para el sistema evaluar.
