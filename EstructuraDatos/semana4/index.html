  • Simular el proceso de llegada y atención en Python.
  • Traducir el principio FIFO a una justificación de equidad operativa.

Teoría aplicada a negocios

La teoría se construye con base en literatura clásica de estructuras de datos y se traduce al lenguaje operativo de NeuroBiz.

1. Cola como estructura de servicio

Cairó Battistutti y Guardati Bueno (2006) presentan la cola como una estructura lineal donde la inserción ocurre en un extremo y la extracción en el otro. Esa separación de extremos refleja de manera natural procesos de espera. En lenguaje de negocios, la cola formaliza una regla de trato: primero en entrar, primero en salir.

En NeuroBiz, la lista de espera de pacientes para asignación de terapeuta exige precisamente esa lógica. Cuando el criterio organizacional es la equidad temporal, la cola no es un detalle técnico sino una forma de traducir una política de servicio a una estructura operativa verificable.

2. FIFO y justicia de proceso

Joyanes Aguilar (2014) explica que la cola responde al patrón FIFO (First In, First Out). La relevancia didáctica está en que el estudiante compare este comportamiento con LIFO y entienda que la estructura depende del criterio del proceso. Aquí no importa lo más reciente, sino el orden de llegada.

FrentePrimer elemento listo para salirPaciente que debe ser atendido primero
FinalExtremo donde se agregan nuevos elementosNuevo paciente que acaba de registrarse
FIFOFirst In, First OutRespeto del orden de llegada

3. Operaciones fundamentales y lectura organizacional

Las operaciones esenciales son enqueue (agregar al final), dequeue (retirar del frente), peek (consultar el frente sin extraer) e isEmpty (verificar si no hay elementos). Weiss (2014) destaca que estas operaciones modelan sistemas de atención y procesamiento de tareas, lo que las vuelve especialmente pertinentes en contextos administrativos.

OperaciónSentido computacionalTraducción organizacionalComplejidad esperada

dequeueExtraer del frenteAsignar el siguiente turno o terapeutaO(1)

4. Representación práctica en Python con deque

La biblioteca estándar de Python ofrece collections.deque, una estructura de doble extremo eficiente para agregar y retirar elementos en ambos lados. Python Software Foundation (2024) documenta su uso como una herramienta adecuada para colas. En esta semana se utiliza deque para mantener el foco en la lógica FIFO sin distraer al estudiante con detalles de desplazamiento interno de listas.

Desde la didáctica, esta decisión es importante: el lenguaje debe facilitar la comprensión del patrón organizacional, no convertirse en una barrera accidental. Aun así, el docente debe verbalizar que la estructura elegida representa la política de servicio, no simplemente una función conveniente del lenguaje.

5. Aplicaciones organizacionales y posibles extensiones

Una cola básica protege la equidad por llegada. Más adelante, si la institución define criterios de urgencia clínica, podrá migrarse hacia colas con prioridad. Sin embargo, conviene primero consolidar el caso FIFO puro. Moreno Pérez (2012) subraya que una buena solución parte de un problema claramente delimitado; mezclar reglas muy pronto puede confundir la comprensión base.

Cuando la decisión central es respetar turnos, la cola no solo organiza datos: hace visible la política de atención de la organización.

Bloque 1: Cola básica con collections.deque colab_queue_deque.py 
from collections import deque

                    cola_espera = deque()

                    def registrar_paciente(cola, nombre, necesidad):
                        cola.append({"paciente": nombre, "necesidad": necesidad})
                        print(f"Ingresó {nombre}. Cola actual: {[item['paciente'] for item in cola]}")

                    def atender_siguiente(cola):
                        if not cola:
                            print("No hay pacientes en espera")
                            return None
                        paciente = cola.popleft()
                        print(f"Se atiende a {paciente['paciente']}")
                        return paciente

                    registrar_paciente(cola_espera, "Samuel", "Lenguaje")
                    registrar_paciente(cola_espera, "Valentina", "Integración sensorial")
                    registrar_paciente(cola_espera, "Martín", "Seguimiento TEA")

                    print("
Paciente al frente:", cola_espera[0])
                    atender_siguiente(cola_espera)
                    print("Cola restante:", [item['paciente'] for item in cola_espera])
Uso pedagógico: Muestra de forma inmediata cómo se registra y atiende una lista de espera preservando el orden de llegada.
Bloque 2: Simulación de asignación de terapeutas con cola de espera colab_queue_assignment.py 
from collections import deque

                    pacientes = deque([
                        {"paciente": "Samuel", "servicio": "Lenguaje"},
                        {"paciente": "Valentina", "servicio": "Sensorial"},
                        {"paciente": "Martín", "servicio": "TEA"},
                        {"paciente": "Lucía", "servicio": "TDAH"},
                    ])

                    terapeutas_disponibles = ["Laura", "Carlos"]
                    asignaciones = []

                    while pacientes and terapeutas_disponibles:
                        paciente = pacientes.popleft()
                        terapeuta = terapeutas_disponibles.pop(0)
                        asignaciones.append((paciente["paciente"], terapeuta, paciente["servicio"]))
                        print(f"Asignado {paciente['paciente']} a {terapeuta}")

                    print("
Asignaciones del bloque:")
                    for asignacion in asignaciones:
                        print(asignacion)

                    print("Pacientes aún en cola:", [p["paciente"] for p in pacientes])
Uso pedagógico: Integra la cola con una necesidad de negocio: asignar pacientes al siguiente recurso disponible.
Bloque 3: Panel operativo para inspeccionar la cola colab_queue_dashboard.py 
from collections import deque

cola = deque([
    {"paciente": "Samuel", "espera_min": 12},
    {"paciente": "Valentina", "espera_min": 18},
    {"paciente": "Martín", "espera_min": 25},
])

def resumen_cola(cola_actual):
    if not cola_actual:
        return {
            "longitud": 0,
            "frente": None,
            "espera_promedio": 0,
        }

    espera_promedio = sum(item["espera_min"] for item in cola_actual) / len(cola_actual)
    return {
        "longitud": len(cola_actual),
        "frente": cola_actual[0]["paciente"],
        "espera_promedio": round(espera_promedio, 2),
    }

print(resumen_cola(cola))
cola.popleft()
print(resumen_cola(cola))
Uso pedagógico: Ayuda a convertir la estructura en una salida legible para la toma de decisiones.

¿Qué significa FIFO en una cola?
