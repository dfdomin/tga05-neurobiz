1. Recursividad directa e indirecta

La recursividad directa ocurre cuando una función se llama a sí misma de manera explícita. Joyanes Aguilar (2014) la usa para introducir el patrón básico de reducción del problema. Es útil porque la relación entre problema grande y problema pequeño se ve inmediatamente.

La recursividad indirecta aparece cuando la llamada pasa por una o más funciones intermedias. Cairó Battistutti y Guardati Bueno (2006) la presentan como una extensión del mismo principio: el proceso sigue siendo recursivo aunque la referencia no sea literal dentro del mismo cuerpo de función.

2. Recursión vs iteración: no es una guerra, es una decisión

Weiss (2014) recuerda que una buena estructura de datos siempre debe analizarse junto con el costo de las operaciones. En clase, esto significa que la pregunta no es “¿cuál técnica es más bonita?”, sino “¿cuál explica mejor el problema y cuál consume menos pasos para este caso?”.

La iteración suele ser más familiar y puede ser más eficiente en tareas lineales sencillas. La recursividad, por su parte, suele comunicar mejor problemas que se dividen naturalmente, como una búsqueda binaria o un ordenamiento tipo Merge Sort.

3. Búsqueda binaria recursiva en historiales ordenados

Si las sesiones de un paciente están ordenadas por fecha o por identificador, no es necesario revisar una a una. La búsqueda binaria reduce el espacio de búsqueda a la mitad en cada paso. Esa reducción es un ejemplo perfecto de aplicación recursiva con valor empresarial claro.

Moreno Pérez (2012) insiste en que el análisis paso a paso es clave para que el estudiante no confunda “dividir el rango” con “moverse secuencialmente”. Por eso conviene contar pasos y registrar el centro evaluado en cada llamada.

4. Programas recursivos como soluciones verificables

Un programa recursivo bien diseñado debe ser demostrable: el estudiante debe poder describir la entrada, el caso base, la reducción del problema y la salida. Ese marco será el mismo cuando más adelante se estudien algoritmos de ordenamiento o estructuras dinámicas.

En el contexto NeuroBiz, la comparación entre búsqueda lineal y binaria no solo mejora la sintaxis; mejora la lectura crítica de procesos organizacionales: si los datos están preparados, el análisis puede ser más rápido y sostenible.

Bloque 1 — Recursión directa e indirecta en validación de perfil

Permite mostrar que la idea recursiva puede distribuirse entre varias funciones del proceso.

Google Colab · Bloque 1Abrir Colab
def validar_perfil_base(perfil):
    return "diagnostico" in perfil and "sesiones" in perfil

def validar_diagnostico(perfil):
    if perfil["diagnostico"] in {"TDAH", "TEA", "ACCC"}:
        return validar_perfil_base(perfil)
    return False

def validar_perfil(perfil):
    if not perfil:
        return False
    return validar_diagnostico(perfil)

perfil_neurobiz = {"diagnostico": "TEA", "sesiones": 12}
print(validar_perfil(perfil_neurobiz))

Bloque 2 — Búsqueda binaria recursiva en sesiones ordenadas

El bloque usa una lista ordenada por id o fecha y registra cada paso para volver visible la reducción del rango.

Google Colab · Bloque 2Abrir Colab
sesiones_ordenadas = [101, 105, 109, 120, 132, 140, 155]

def busqueda_binaria_recursiva(lista, objetivo, inicio, fin, pasos=0):
    if inicio > fin:
        return -1, pasos
    pasos += 1
    medio = (inicio + fin) // 2
    if lista[medio] == objetivo:
        return medio, pasos
    if objetivo < lista[medio]:
        return busqueda_binaria_recursiva(lista, objetivo, inicio, medio - 1, pasos)
    return busqueda_binaria_recursiva(lista, objetivo, medio + 1, fin, pasos)

posicion, pasos = busqueda_binaria_recursiva(sesiones_ordenadas, 132, 0, len(sesiones_ordenadas) - 1)
print(f"Encontrada en posición {posicion}, pasos {pasos}")

Bloque 3 — Comparación lineal vs binaria midiendo pasos

El objetivo es analizar, no memorizar: el estudiante debe explicar por qué la segunda estrategia usa menos verificaciones cuando la lista está ordenada.

Google Colab · Bloque 3Abrir Colab
def busqueda_lineal(lista, objetivo):
    pasos = 0
    for indice, valor in enumerate(lista):
        pasos += 1
        if valor == objetivo:
            return indice, pasos
    return -1, pasos

objetivo = 155
pos_l, pasos_l = busqueda_lineal(sesiones_ordenadas, objetivo)
pos_b, pasos_b = busqueda_binaria_recursiva(sesiones_ordenadas, objetivo, 0, len(sesiones_ordenadas) - 1)

print(f"Lineal: posición={pos_l}, pasos={pasos_l}")
print(f"Binaria recursiva: posición={pos_b}, pasos={pasos_b}")
