📚 Marco Teórico

1. ¿Qué es una Estructura de Datos?

Una estructura de datos es una forma organizada de almacenar y gestionar información en la memoria de un computador, de modo que pueda ser accedida y modificada de manera eficiente (Cairó & Guardati, 2006). Para NeuroBiz, elegir la estructura incorrecta puede significar buscar el historial de un paciente en segundos… o en horas.

💡

Analogía empresarial: Un cajón de archivos es una estructura de datos física. Si los archivos están en orden alfabético (arreglo ordenado), encontrar el expediente de un paciente es rápido. Si están en desorden, toca revisar uno por uno.

2. Tipos de Datos en Python

Según Joyanes Aguilar (2014), los tipos de datos son la base de toda estructura. Python clasifica los datos en:

🧠

NeuroBiz + Neurodiversidad: Las personas con TDAH suelen procesar mejor la información visual y en bloques cortos. Por eso NeuroBiz almacena el nivel_hiperfoco (float) para personalizar la duración de cada sesión. Un dato bien tipado permite decisiones más humanas.

3. Almacenamiento en Memoria

Cuando Python ejecuta x = 847, reserva un bloque de memoria con una dirección, un tipo y un valor. Weiss (2014) señala que la eficiencia de un algoritmo depende directamente de cómo se asigna y libera esa memoria.

En Python, la gestión es automática (garbage collector), pero el analista debe elegir el tipo correcto para evitar desperdicio: guardar 847 como str en lugar de int impide hacer operaciones matemáticas directamente.

⚠️

Un error frecuente en bases de datos empresariales: importar todos los campos como texto (str). Esto obliga a convertir cada dato antes de calcularlo, generando errores silenciosos en los reportes.

4. Arreglos (Arrays) — La estructura más básica

Un arreglo es una colección de elementos del mismo tipo almacenados en posiciones contiguas de memoria, accesibles mediante un índice (Cairó & Guardati, 2006).

En Python puro no existe el arreglo tradicional de Java/C; se usan list o, para rendimiento profesional, numpy.array. Para análisis de datos en NeuroBiz usaremos listas como punto de partida.

OperaciónDescripciónComplejidad
Acceso por índicearr[2]O(1) — Instantáneo
Búsqueda linealRecorrer elemento por elementoO(n)
Inserción al finalarr.append(x)O(1) amortizado
Inserción en medioarr.insert(i, x)O(n)
Eliminaciónarr.remove(x)O(n)
🏢

Contexto NeuroBiz: Si almacenamos en un arreglo las 847 sesiones del mes, acceder a la sesión número 200 es inmediato (O(1)). Pero buscar todos los pacientes con tasa_retencion < 0.5 requiere recorrer todo el arreglo (O(n)). Esta diferencia importa cuando escalamos a 10.000 sesiones.

Bloque 2 — Arreglo de sesiones mensuales

📄 arreglo_sesiones.py
# ── Arreglo: historial de sesiones por semana ────────────
sesiones_semanales = [198, 215, 202, 232]  # 4 semanas de enero

# Acceso directo (O(1))
print(f"Semana 3: {sesiones_semanales[2]} sesiones")

# Recorrido completo (O(n))
print("\n📅 Detalle mensual:")
for i, ses in enumerate(sesiones_semanales, 1):
    estado = "✅ CUMPLE" if ses >= 200 else "⚠️ BAJO META"
    print(f"  Semana {i}: {ses:3d} sesiones — {estado}")

# Estadísticas básicas (BI)
print(f"\n📈 Total  : {sum(sesiones_semanales)}")
print(f"   Máximo : {max(sesiones_semanales)}")
print(f"   Mínimo : {min(sesiones_semanales)}")
print(f"   Media  : {sum(sesiones_semanales)/len(sesiones_semanales):.1f}")
🎯

¿Qué acabas de construir? Un micro-dashboard en consola. En módulos avanzados, este mismo código alimentará visualizaciones con matplotlib y pandas. La lógica es idéntica; solo cambia la capa de presentación.

🎮 Quiz de Verificación — Quizizz

5 preguntas · 8 minutos · Modo Playground

Pasa el cursor sobre cada pregunta para ver la respuesta correcta (modo estudio). En clase el docente lanzará el quiz en vivo.

1. En NeuroBiz, sesiones_asistidas = "847" está declarado como str. Si intentas calcular sesiones_asistidas + 10, Python…

  1. Devuelve 857
  2. Genera un TypeError
  3. Devuelve "84710"
  4. Redondea al entero más cercano
✅ b) Genera un TypeError — no se puede sumar str + int directamente.

2. ¿Qué operación sobre un arreglo de 1.000 sesiones es O(1)?

  1. Buscar la sesión con mayor índice de progreso
  2. Eliminar el primer elemento
  3. Acceder a sesiones[499]
  4. Recorrer todo el arreglo para calcular el promedio
✅ c) Acceder por índice es O(1) — directo a la posición de memoria.

3. ¿Cuál tipo de dato en Python es el más apropiado para almacenar tasa_retencion = 0.78?

  1. int
  2. str
  3. float
  4. bool
✅ c) float — los porcentajes decimales requieren punto flotante.

4. NeuroBiz necesita saber si una sesión fue completada (sí/no). ¿Qué tipo es más eficiente?

  1. int con valor 0 o 1
  2. bool con True o False
  3. str con "si" o "no"
  4. Cualquiera es igual de eficiente
✅ b) bool — semánticamente correcto y usa menos memoria que str.

5. ¿Cuál es la principal ventaja de un arreglo frente a variables separadas para las 847 sesiones?

  1. Las variables separadas son más rápidas de acceder
  2. El arreglo permite tratar todos los datos de forma homogénea con un índice
  3. El arreglo siempre ocupa menos memoria
  4. Las variables separadas permiten más tipos de datos
✅ b) Organización homogénea e indexada — esencial para procesamiento masivo.
